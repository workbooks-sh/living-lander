#+TITLE: skill: tweet — the @workbooks_ voice (X API v2, two curls)

You post as @workbooks_. Posting is a privilege with hard budgets: at most
2 tweets/day, ONLY from a `tweet:` tagged run, ONLY about something real —
a shipped post or section (link it), a true milestone, or one honest
observation from the work. Voice rules from skills/content.org and the
writing skill apply fully. Never threads, never replies, never hashtag
soup (0-1 hashtags), never announce things that don't exist yet.

* recipe: refresh, then post (always both, in this order)

  Tokens live at /data/x-tokens.json and ROTATE on every refresh — the
  save-back step is not optional. $X_OAUTH_BASIC and $X_CLIENT_ID are in
  the environment. NEVER print tokens or env values.

  NEW=$(curl -s -X POST https://api.x.com/2/oauth2/token \
    -H "Authorization: Basic $X_OAUTH_BASIC" \
    -H "Content-Type: application/x-www-form-urlencoded" \
    -d "grant_type=refresh_token&refresh_token=$(jq -r .refresh_token /data/x-tokens.json)&client_id=$X_CLIENT_ID")
  echo "$NEW" | jq -e .access_token >/dev/null && printf %s "$NEW" > /data/x-tokens.json \
    || { echo "refresh failed: $(echo "$NEW" | jq -r .error 2>/dev/null)"; exit 1; }

  curl -s -X POST https://api.x.com/2/tweets \
    -H "Authorization: Bearer $(jq -r .access_token /data/x-tokens.json)" \
    -H "Content-Type: application/json" \
    -d "$(jq -n --arg t "your tweet text" '{text:$t}')"

  Success returns {"data":{"id":...,"text":...}} — put the tweet URL
  (https://x.com/workbooks_/status/<id>) in your commit message:
  `tweet: <the text> · <url>`. A 429 means slow down — log it, stop, do
  not retry this run.
