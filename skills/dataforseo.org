#+TITLE: skill: dataforseo — real search data (volumes, SERPs, rankings)

You have a live DataForSEO account. Auth is in the environment as
$DATAFORSEO_AUTH — NEVER print it, NEVER write it into any file. Every call
costs real money (fractions of a cent) — one or two calls per blog run, not
exploratory loops. All endpoints: POST JSON arrays to api.dataforseo.com/v3.

* recipe: keyword search volume (use in the blog query gate)

  Validate your target query before writing — pick the candidate people
  actually search:

  curl -s -X POST "https://api.dataforseo.com/v3/keywords_data/google_ads/search_volume/live" \
    -H "Authorization: Basic $DATAFORSEO_AUTH" -H "Content-Type: application/json" \
    -d '[{"keywords":["build internal tools without developers","single file web app"],"location_code":2840,"language_code":"en"}]'

  Read tasks[0].result[].search_volume (+ cpc, competition). Prefer 50+
  monthly volume; below 10, choose a different query unless the post is
  proof-content with another reason to exist.

* recipe: live SERP — who already ranks (shape your angle)

  curl -s -X POST "https://api.dataforseo.com/v3/serp/google/organic/live/regular" \
    -H "Authorization: Basic $DATAFORSEO_AUTH" -H "Content-Type: application/json" \
    -d '[{"keyword":"build internal tools without developers","location_code":2840,"language_code":"en","depth":10}]'

  Read tasks[0].result[0].items[]: title, url, description of the top 10.
  Your post must say something the top 10 don't — note the gap in your plan
  line before writing.

* recipe: what this site already ranks for (audit runs, occasionally)

  curl -s -X POST "https://api.dataforseo.com/v3/dataforseo_labs/google/ranked_keywords/live" \
    -H "Authorization: Basic $DATAFORSEO_AUTH" -H "Content-Type: application/json" \
    -d '[{"target":"workbooks.sh","location_code":2840,"language_code":"en","limit":20}]'

  Early on this returns little — that's honest data, log it and move on.

* budget rules

  - strategy runs (landscape research): exploratory loops are FINE — up to
    ~12 calls; map a whole cohort's queries and SERPs properly
  - blog/section ADD run: ≤2 calls (volume gate + optional SERP scan)
  - audit run: ≤1 call (ranked keywords, every few audits)
  - never call it from daydreams/dreams
