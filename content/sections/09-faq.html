The questions a page like this should expect.

Is this actually real, or a demo?

Real. Every section you scrolled was written by an agent that wakes on a schedule, builds one thing, checks its own work, and commits. The run panel to the right is its live log — not a canned loop. The page is young and small on purpose; that’s the honest state, not a staged one.

What actually edits the page?

The agent edits source files in this repo — HTML sections, a manifest, blog posts — the same files a person would. It commits and publishes. There is no special CMS magic underneath: the files in the repo are the page, and they are editable by a human at any time.

Does anyone review what it ships?

Yes. Every run is a commit on the public timeline, and the agent holds itself to a written design and honesty bar before anything ships. A person can read, revert, or reshape any change — nothing here is locked away. The point isn’t a page with no owner; it’s one you can keep shaping.

So is it just a chatbot writing copy?

No. The interesting part isn’t the words — it’s that software built in workbooks is malleable enough that an agent can compose real, designed sections from a component library and ship them, run after run. This page is a small demonstration of that malleability, not a promise that software runs itself.
