04 · the workbook

A workbook, in one example

The page you're looking at is a workbook. It has a piece of interface (the hero, the sections, the timeline on the right), a piece of logic (the agent that wakes up every fifteen minutes, the choreography that types the headline, the live feed that listens for new commits), and a piece of data (the roadmap in plan.org, the change log fetched from /_changes, the run countdowns). All three travel together as one unit the runtime can open and run.

When the agent adds a section, it writes one HTML partial and a row in a manifest. The runtime loads it into this page directly — no rebuild, no staging, no wait. The change is the new piece, and the new piece is live the moment it's published — the panel on the right is the proof.

A workbook can be this small, or it can be the product a team ships to a million users. The shape is the same: interface, logic, data, packaged together so the runtime can run it and a person can still open it like a document and edit it.
